CE 错误总结

摘选了一些比较有代表性的错误。
  • 静态数组的大小必须在编译时已知。变量一般得有个名字。以下代码中数组g的和某大数组的声明不合法。 
    struct M{
	int f,fa,fb,fc,fd;
	int g[fd];
}[100005];
    这还引出一个问题，g++编译时，局部数组的大小是可以用变量的。但这其实是非法的行为，不可移植。
  • 忘加分号。（一份输出样例的代码因此编译失败
  • 没有声明变量就使用，或是变量不可达。(这不是python或是js！！
  • 函数实参和形参不匹配（数量，类型），比如fclose("julian.in",stdin);来自一份打表代码，不然能有点分。
  • 超大数组（即便题目没有空间限制，计算机内存也是有限的...过大无法寻址）int a[100000000000000];
  • 非数组当成数组用...一维数组当二维用...
  • 提前结束了块
    for (int i = 0; i < Q; i++) {
  cin >> r[i];
}
if (r[i] == 10) {
  • 括号不匹配
  • main返回值必须是int。long long main() {
  • 命名冲突。如同时using namespace std和使用next做变量名...
  • 手滑：if(n==2&&&m==2&&c==4&&k==3)
  • 不存在的类型转换。srand(int(new char));首先使用new出来的地址做随机种子这种行为是危险且不负责的。建议开long long转换。c风格类型转换也是不推荐的。
  • 定义类型后要加分号
  • == 和 =
  • NULL为0，而null不存在。srand(time(null) - 1919810 + 114514 + 19260817);
  • time函数接收的是一个指向time_t类型对象的指针。srand(time(19260817));
